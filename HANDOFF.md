# Handoff Notes

이 문서는 다른 데스크탑/다른 에이전트가 이 저장소를 바로 이어서 작업할 수 있도록 현재 상태를 정리한 인수인계 문서입니다.

## 저장소 개요

- 저장소: `wani3000/mywedding`
- 브랜치: `main`
- 배포: GitHub Pages
- 메인 페이지: `wedding_guide.html`
- 진입점: `index.html` -> `wedding_guide.html`로 리다이렉트
- 현재 헤더 날짜 표기: `2026년 5월 2일 (토) · 12:30 · Noble Valenti 대치점`

## 파일 역할

- `wedding_guide.html`
  - 실제 웨딩 안내 페이지
  - CSS / HTML / JS가 모두 한 파일 안에 들어 있는 정적 페이지
- `index.html`
  - GitHub Pages 루트 진입용
- `README.md`
  - 간단한 저장소 설명
- `HANDOFF.md`
  - 현재 문서

## 현재 탭 구조

상단 탭은 아래 3개입니다.

- `식순`
- `준비사항`
- `대본`

해시 기반 탭 동작:

- `#schedule` -> 식순
- `#prep` -> 준비사항
- `#script` -> 대본

초기 진입 탭:

- 해시가 없으면 `식순`

관련 위치:

- 탭 네비게이션: `wedding_guide.html` 상단 `nav.top-tabs`
- 탭 전환 스크립트: 파일 하단 `setActiveTab`

## 식순 탭 현재 내용

`식순` 탭에는 아래가 들어 있습니다.

- `⏱ 당일 전체 타임라인`
- `👨‍👩‍👧 혼주 및 진행 인원`

현재 주요 타임라인 기준:

- 식 시작: `12:30`
- 축사: `12:48`, `약 2분 30초`
- 축가: `12:51`
- 부모님 감사인사: `12:54`
- 행진: `12:58`
- 폐회: `13:00`
- 원판 및 연출 촬영: `13:01`
- 피로연 인사: `13:31`
- 정산: `14:01~`

주의:

- 사용자가 “축사만 늘리고 뒤 일정을 그만큼 뒤로 미는 방식”을 원해서 현재 시각이 그렇게 반영되어 있음
- 현재 안내문도 `축사 2분 30초` 기준으로 맞춰져 있음

## 준비사항 탭 현재 내용

`준비사항` 탭에는 아래 섹션이 들어 있습니다.

- `📝 식 전 준비 체크리스트`
- `📎 파일 및 보증 안내`
- `📋 웨딩 진행 사항`
- `🏷 외부업체 및 촬영 정보`
- `🚗 주차 및 교통`
- `💳 정산`
- `✅ 체크사항`

### 체크리스트 상태

현재 체크 상태는 아래와 같습니다.

보내야 하는 파일:

- `식순 (사회자 대본)` 체크됨
- `식전 영상` 체크됨
- `축가MR` 체크됨
- `성혼선언문` 체크됨
- `주례사 낭독문` 체크됨
- `축사 낭독문` 체크됨
- `모든 파일 4/28 (화)까지 메일 발송` 체크됨

당일 가져갈 물품:

- `식권` 미체크
- `대형 액자 1개 준비 여부 확인` 체크됨
- 나머지 물품은 미체크

### 식권 관련

`당일 가져갈 물품`에 아래 문구가 들어가 있음:

- `식권 주최측(직접) 준비`
- `식권에 도장 및 싸인 그리고 넘버링 작업 필수입니다!`

### 식전 영상 관련

체크리스트의 `식전 영상`에는 아래 문구가 들어가 있음:

- `진행 (주최측 준비)`
- `식 20분전~10분전까지 웨딩홀 내 중앙 스크린에서 상영`

### 구글드라이브 링크

체크리스트 바로 아래에 아래 링크가 들어가 있음:

- `https://drive.google.com/drive/folders/1xRWqU5iYAFhElWR9rhWtlbEtwyFr7dua?usp=drive_link`

### 준비사항 본문 원칙

사용자는 준비사항 본문에 대해 “원문 표현과 100% 동일하게” 맞추는 것을 강하게 선호함.

특히 아래 표현들은 원문 기준으로 맞춘 상태임:

- `엠프연결선`
- `의류&액자&포토테이블 사진 등등은 사전 보관 어려우니...`
- `당사의 제휴업체를 제외한 외부 업체 및 물품 반입은...`

새로 수정할 때는 체크리스트 요약 문구와 `본문 원문`을 섞어 망가뜨리지 않도록 주의.

## 대본 탭 현재 내용

`대본` 탭은 카드형이 아니라 PDF 스타일을 참고한 표 형식입니다.

구조:

- 3열 표
- 열 제목: `진행순서 / 안내문 / 체크사항`

관련 스타일:

- `.script-table`
- `.script-step`
- `.script-guide`
- `.script-check`

### 대본 내용 상태

현재 대본은 사용자가 전달한 PDF/이미지 기준 내용으로 맞춰져 있음.

중요 반영 사항:

- 이름은 파란색 굵게 표시
- 축사 시작 시 `(축사 시작할 때 연주 요청?)` 문구는 취소선 처리
- 같은 줄에 `연주요청 하지 않음` 추가
- 부모님 감사 인사 구간의 포옹 대상 순서 수정 완료

### 최근 식순/오디오 메모 반영

최근 다른 에이전트가 추가한 내용:

- 화동 입장 줄 아래 `배경음악.mp3`
- 축사 줄 아래 `피아노반주 예정`

둘 다 `식순` 탭의 타임라인 보조 문구로 들어가 있음.

주의:

- 사용자는 “형식보다 내용 일치”를 더 중요하게 확인했음
- 하지만 대본 표는 PDF처럼 보이도록 어느 정도 맞춰 둔 상태

## 배포 및 검증 방식

일반 흐름:

1. `wedding_guide.html` 수정
2. `git status`
3. `git add ...`
4. `git commit -m "..."`
5. `git push origin main`
6. GitHub Pages 응답 확인

자주 확인한 URL:

- 배포 루트: `https://wani3000.github.io/mywedding/`
- 직접 파일: `https://wani3000.github.io/mywedding/wedding_guide.html`

참고:

- 모바일 브라우저 캐시 때문에 방금 반영한 내용이 바로 안 보일 수 있음
- 이 경우 새로고침 또는 탭 재오픈이 필요할 수 있음

## 다음 작업자가 주의할 점

- 사용자는 짧게 말하지만 디테일 수정이 매우 많고, 원문 표현 차이에 민감함
- “커밋푸시배포해” 요청이 매우 자주 나오므로, 수정 후 바로 푸시 가능한 상태를 유지하는 것이 좋음
- 준비사항 본문은 원문 유지
- 체크리스트는 실행용 요약
- 대본은 표 구조 유지
- 타임라인은 사용자의 최신 진행 시간 기준을 우선

## 빠른 점검 체크

다음 작업 전/후에 빠르게 보면 좋은 항목:

- `식순 / 준비사항 / 대본` 탭이 각각 정상 동작하는지
- `준비사항` 탭에서 체크리스트가 보이는지
- `대본` 탭에서 표가 깨지지 않았는지
- `식순` 탭에서 축사 시간이 `2분 30초`인지
- `폐회`가 `13:00`인지
- `축가MR` 체크 상태가 유지되는지
- `축사 낭독문`, `모든 파일 발송`이 체크 상태인지
- 화동 `배경음악.mp3`, 축사 `피아노반주 예정`이 보이는지
